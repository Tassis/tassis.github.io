Postgresql and PgAdmin4 Installation on Debian9

發表於 2019-02-10 | 更新於 2019-04-25 | 分類於 Database ， PostgreSQL

PostgreSQL 是近兩年在社群上跛為熱門的資料庫系統，雖然在台灣較沒有人聽過，但是憑著其穩定性及工具集的方便性，被越來越多的人採用。而 PgAdmin 則是 PostgreSQL 的操作 GUI界面，基於 Flask 框架構成，地位上有些類似 phpMyAdmin 但功能更多，由於 VPS 上正好需要資料庫系統，因此在這邊順便紀錄下自己的安裝過程。

前言

要完全部署完畢並不單單只有安裝 PostgreSQL 及 PgAdmin 而已 ，由於 pgAdmin 是一套基於 Flask 框架的 web gui ，因此需要先準備好對應的運行環境。 具體需求如下：

  • Htttp Server (ex: apache 、 nginx )
  • WSGI Server (ex: Gunicorn、uWSGI)
  • Python Enviourment

考量到未來 linode 上可能會有其他的 python 實驗環境，因此這邊採用 virtualenv 額外建置虛擬環境來做運行。若不知道 virtualenv 的用途的話可以參考 Install Pyenv and virtualenv on Debian9 這篇。

而 HttpServer 的部份由於個人偏好選用 Nginx ， WSGI Server 採用 Flask 框架部署時常用的 uWSGI 。

uWSGI 官方手冊（簡體中文翻譯）: 傳送門

上面是這次所編寫的範例，針對出現的變數大致講解下代表什麼意思。

  • process ，啟動時會運行多少個進程（數量依據需求而定）。
  • threads ， prethreaded模式下，每個進程會使用多少執行緒。
  • socket ，啟動時會在指定路徑生成對應 sock 檔案，用來與 http 伺服器 (nginx) 溝通的主要手段。
  • chdir ， 程式運行前先切換到指定目錄，
  • chmod-socket 變更生成 socket 檔案的預設權限 ( 確保 http server 有權限可訪問 一般設定為 644 )
  • mount ， 在檔案內加載程式，這邊將其指向到 pyAdmin4.py 中的 app變數。
  • wsgi-file ， 運行時所需要的協議檔案。

  • User 請改為自己希望執行 uWSGI 的使用者
