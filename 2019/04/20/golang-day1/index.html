Golang 初學筆記（一） on Ubuntu 16.04

發表於 2019-04-20 | 分類於 程式語言 ， golang

研究 Golang 有一段時間，多少有些心得。 目前操作起來與其他的程式語言有許多不同的地方，還在一步一步的研究中，因此在這邊紀錄下遇到的一些問題，希望未來能夠有些參照對應。

初始化 Golang 專案

以往撰寫 Golang專案需要將專案放置在 GOPATH 環境變數下（或者將專案資料夾放入 GOPATH 內部）才能夠正常的讀取套件，但於 1.11 版後提供了 go mod 指令，用於初始化專案，可使專案不用在放置於 GOPATH 內部。

導入本地端 Package

Golang 的 import 機制依賴於 GOPATH ， 系統會自動去查找 GOPATH 下是否有相同名稱的 package 若沒有的話則會顯示：

Can’t find module providing package.

因此倘若在專案中存在著自己撰寫的幾個子套件並且未放置於 GOPATH 下的話，會無法 import 入專案內，需要另外在 go.mod 檔案進行配置，內容如下：

先用運 require 設定依賴的 package ，再透過 replace 手動指定 package 的路徑到想要的位置。 這樣若專案內有好幾個 sub package 便可以透過設定導入進來。

VSCode 的編譯快捷設定
